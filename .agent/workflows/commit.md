---
description: 변경 사항 분석, 한국어 커밋 메시지 제안, Husky 검증을 거친 안전한 커밋 및 푸시
---

1. `git status`를 실행하고, 사용자에게 스테이징할 파일을 묻는다. 답변에 따라 `git add`를 실행한다.

2. `git diff --staged`로 변경 내용을 분석한다.

3. 분석 내용을 바탕으로 **한국어 커밋 메시지 옵션 3개**를 생성하여 제시한다.
   - 포맷: `[type]: [description]` (예: `feat: 로그인 기능 추가`)

4. **사용자 선택 단계**:
   - 사용자에게 "위 3가지 중 어떤 메시지를 사용하시겠습니까?"라고 묻는다.
   - 사용자가 번호를 선택하면 해당 메시지로, 직접 입력하면 입력한 내용으로 커밋 메시지를 확정한다.

5. `git commit -m "[확정된 메시지]"`를 실행한다.

   > **참고**: 커밋 시 `Husky(pre-commit)`가 자동으로 실행되며, 실패 시 사용자에게 알린다.

6. 사용자에게 "푸시(Push) 진행 여부"를 묻는다. 긍정적인 답변(`y`, `yes` 등)을 받은 경우에만 `git push`를 실행하고, 그렇지 않으면 종료한다.
